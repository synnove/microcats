Part I

Building a Wireless Sensor Network

A Wireless Sensor Network (WSN)

  • Consists of multiple sensor nodes
  • Communicating wirelessly
  • With a gateway
  • That aggregates data from the sensors
  • And sends it on

Network Topologies

Why WSNs?

  • Nodes are cheap and require low power
  • Little management required
  • Collect lots of data in short intervals
  • Collect lots of data across a wide area

What we're working with

  • 4x Waspmotes
  • 1x Meshlium
  • 1x uqcloud zone

Waspmotes

  • ATmega1281 microcontroller
  • Zigbee radio
  • Solar panels!

They're equipped with sensors

  • Temperature
  • Humidity
  • CO
  • CO2
  • O3
  • With these we can measure pollution levels

Meshlium

  • Modified Debian
  • Zigbee radio
  • Wifi AP
  • Ethernet connection
  • No SSH access..

uqcloud zone

  • Ubuntu 14.04.3 Trusty Tahr
  • Nginx web server
  • PostgreSQL database
  • MQTT broker

Part II

Online Real-time Microclimate Analysis

Online Real-time Microclimate Analysis

  • Having raw data is good for analysis and number-crunching
  • We also want to do interesting things with the data collected
  • Visualising data makes it easier to understand it at a glance

But..

  • Creating useful visualisations is difficult
  • Important to consider how to present data
  • Without overwhelming or confusing the viewer

D. Hoang, et al., ”Processing and Visualizing Traffic Pollution Data in Hanoi City from a Wireless Sensor Network”, 2013 IEEE 38th Conference on Local Computer Networks Workshops (LCN Workshops), 2013.
